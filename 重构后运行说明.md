# 重构后运行说明

本文档汇总当前非法律主线的运行命令。当前主线包括：

```text
user_profile_project_v2           用户画像离线清洗、SVM 模型、标签生成
user_profile_visualization        用户画像后端接口和 Vue 前端页面
user_profile_scripts              WSL 一键运行入口
```

法律相关目录暂不参与当前运行链路。

## 1. 为什么在 WSL/虚拟机中运行

这里说的“虚拟机”主要指 WSL Ubuntu，它不是因为项目只能在远程服务器跑，而是为了提供一个和大数据组件更匹配的 Linux 本地环境。

本项目重构后的计算链路依赖 Spark、Scala、Maven、MySQL 和大量 Linux shell 脚本。虽然源码放在 Windows 的 `D:` 盘，但实际运行推荐放在 WSL 中，原因如下：

1. Spark、Scala、shell 脚本在 Linux 环境下更稳定，路径、权限、脚本行为都更接近真实大数据运行环境。
2. 旧项目本来依赖 CDH、Hive、HDFS、YARN 等 Linux 生态组件，重构后虽然去掉了这些重依赖，但 Spark 本地模式仍更适合在 Linux 里运行。
3. Windows PowerShell 当前没有全局 `mvn`、`npm`，而 WSL 已经配置好 JDK 17、Maven 3.9.6、Scala 2.12、Spark 3.5.1、MySQL 8。
4. 在 Windows 挂载目录 `/mnt/d` 直接做 Maven 构建时，曾遇到 `target` 写入权限问题，所以脚本会复制到 `/home/hadoop/export/workspace` 再构建。
5. 前端 Node 也放在 WSL 用户目录 `/home/hadoop/export/servers`，不需要污染 Windows 全局环境。

因此，推荐理解为：源码在 Windows 本地，运行环境在本机 WSL Linux 中。它仍然是本地运行，不是部署到远程服务器。

## 3. 第一步：运行画像计算链路

打开 WSL 终端，执行：

```bash
cd /mnt/d/code/big_data_case_study/trunk/user_profile_scripts
bash run_user_profile_v2_wsl.sh
```

这个脚本会依次运行：

```text
1. user_profile_project_v2/bin/run_retention_model_wsl.sh
2. user_profile_project_v2/bin/verify_outputs_wsl.sh
```

核心处理流程：

```text
原始 CSV 抽样/读取 -> 离线清洗 -> Parquet 中间表 -> SVM 模型训练 -> 用户挽留预测 -> 用户画像标签 -> MySQL 输出验证
```

默认输入数据：

```text
/mnt/d/BaiduNetdiskDownload/media_index_test
```

默认中间结果：

```text
/home/hadoop/export/data/user_profile_project_v2/processed
/home/hadoop/export/data/user_profile_project_v2/model
```

默认 MySQL 输出表：

```text
big_data_case_study.user_profile_svm_metrics
big_data_case_study.user_profile_svm_prediction
big_data_case_study.user_label
```

## 4. 第二步：启动可视化后端

另开一个 WSL 终端，执行：

```bash
cd /mnt/d/code/big_data_case_study/trunk/user_profile_visualization
bash run_visualization_wsl.sh
```

启动成功后后端地址是：

```text
http://localhost:8001/user_profile
```

注意：这个地址是后端根路径，不是页面入口。直接打开：

```text
http://localhost:8001/user_profile/
```

看到 `Whitelabel Error Page 404` 是正常的，因为后端只提供 API，没有首页。

可以用下面接口验证后端是否正常：

```bash
curl http://localhost:8001/user_profile/api/screen/summary
curl http://localhost:8001/user_profile/api/screen/parent-stats
curl http://localhost:8001/user_profile/api/labels/samples
```

浏览器也可以直接打开：

```text
http://localhost:8001/user_profile/api/screen/summary
```

如果能看到 JSON，说明后端已经正常连上 MySQL。

## 5. 第三步：启动前端页面

第一次运行前端前，建议先构建一次，它会自动准备 WSL Node.js：

```bash
cd /mnt/d/code/big_data_case_study/trunk/user_profile_visualization/frontend-vue
bash build_wsl.sh
```

构建通过后，启动前端开发服务：

```bash
cd /mnt/d/code/big_data_case_study/trunk/user_profile_visualization/frontend-vue
export PATH=/home/hadoop/export/servers/node-v20.19.5-linux-x64/bin:$PATH
npm run dev -- --host 0.0.0.0
```

前端页面地址：

```text
http://localhost:5173/
```

实时大屏地址：

```text
http://localhost:5173/screen
```

前端会通过 Vite 代理访问后端：

```text
/api -> http://localhost:8001/user_profile
```

所以后端 `8001` 要保持运行，前端 `5173` 才能正常展示数据。

## 7. 常见问题

### 7.1 打开 8001 看到 Whitelabel Error Page

原因：`8001` 是后端接口服务，没有前端首页。

正确页面地址：

```text
http://localhost:5173/
```

后端正确验证地址：

```text
http://localhost:8001/user_profile/api/screen/summary
```

### 7.2 PowerShell 中 mvn 或 npm 找不到

当前推荐在 WSL 中运行，不依赖 Windows 全局 Maven/Node。

后端用：

```bash
bash run_visualization_wsl.sh
```

前端用：

```bash
bash build_wsl.sh
export PATH=/home/hadoop/export/servers/node-v20.19.5-linux-x64/bin:$PATH
npm run dev -- --host 0.0.0.0
```

### 7.3 前端页面没有数据

先确认后端 API 是否正常：

```bash
curl http://localhost:8001/user_profile/api/screen/summary
```

如果后端没有返回 JSON，检查：

```text
MySQL 是否启动
bigdata / BigData@123456 是否能连接
user_label 表是否已经由第一步生成
```

### 7.4 需要重新生成画像数据

重新运行：

```bash
cd /mnt/d/code/big_data_case_study/trunk/user_profile_scripts
bash run_user_profile_v2_wsl.sh
```

然后刷新前端页面即可。
